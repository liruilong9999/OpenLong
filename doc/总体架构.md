# OpenClaw 类系统架构设计说明

本文档描述一个精简版 Agentic AI 系统架构，用于指导代码生成工具（如 Codex）逐步生成系统代码。

本版本 **去除多渠道聊天（Channels / Connectors）模块**，只保留 **本地 Web / CLI 入口**，使系统结构更简单、实现成本更低，适合作为第一版架构。

系统核心目标：

1 支持 Agent 智能体运行
2 支持工具调用（Tool Use）
3 支持长期记忆系统
4 支持用户上下文
5 支持 Agent 人格
6 支持插件技能系统
7 支持多模型调用
8 支持 Web Dashboard

------------------------------------------------------------
一、系统总体架构
------------------------------------------------------------

系统划分为 7 个核心模块：

1 Gateway Runtime（系统核心）
2 Agent Runtime（Agent 运行时）
3 Context System（上下文系统）
4 Memory System（记忆系统）
5 Skill System（技能系统）
6 Tool System（工具系统）

------------------------------------------------------------
三、模块设计（详细）

------------------------------------------------------------
1 Gateway Runtime（系统核心）
------------------------------------------------------------

Gateway 是整个系统的控制中心。

职责：

接收用户消息
路由请求到 Agent
管理 Session
管理 Agent 生命周期
调度模型调用
调度工具调用
提供 Dashboard API

核心组件：

API Server
WebSocket Server
Session Manager
Agent Manager
Model Router
Task Queue
Event Bus

说明：

用户所有请求都会先进入 Gateway，然后再交给 Agent Runtime。

------------------------------------------------------------
2 Agent Runtime（Agent 运行时）
------------------------------------------------------------

Agent Runtime 负责执行 Agent Loop。

Agent Loop 是整个系统的核心逻辑。

Agent Loop 流程：

1 接收用户输入
2 构建 Prompt
3 调用 LLM
4 判断是否需要调用 Tool
5 执行 Tool
6 写入 Memory
7 生成回复

核心组件：

Prompt Builder
Planner
Tool Executor
Memory Writer
Response Generator

系统可以运行多个 Agent，例如：

main agent
coding agent
research agent

每个 Agent 都有独立 Workspace。

------------------------------------------------------------
3 Context System（上下文系统）
------------------------------------------------------------

Context 用于构建 Prompt。

包含用户信息和 Agent 人格信息。

组成：

USER.md

内容：

用户背景
用户偏好
沟通风格
常见需求

SOUL.md

内容：

Agent 人格
行为方式
语气风格

IDENTITY.md

内容：

Agent 身份
角色定位

RULES.md

内容：

行为限制
系统规则

STYLE.md

内容：

回复格式
语言风格

Context Manager 负责读取这些文件并构建 Prompt。

------------------------------------------------------------
4 Memory System（记忆系统）
------------------------------------------------------------

Memory 用于长期记忆。

作用：

跨会话记忆
知识积累
事实记录

Memory 类型：

短期记忆
长期记忆
摘要记忆

Memory 结构：

MEMORY.md
memory/logs
memory/summaries

核心组件：

Memory Writer
Memory Retriever
Memory Compressor

功能：

记忆写入
记忆检索
记忆压缩
记忆衰减

------------------------------------------------------------
5 Skill System（技能系统）
------------------------------------------------------------

Skill 是 Agent 的能力插件。

Skill 结构：

skills/
   skill_name/
       SKILL.md
       script(optional)

SKILL.md 内容：

技能描述
使用条件
参数说明
调用示例

作用：

指导 Agent 在什么情况下使用该技能。

Skill 本质：

Prompt 模板 + Tool 调用说明

------------------------------------------------------------
6 Tool System（工具系统）
------------------------------------------------------------

Tool 是系统执行任务的底层能力。

常见 Tool：

shell 执行
文件读写
HTTP 请求
浏览器控制
邮件发送
数据库查询

Tool 执行流程：

Agent 决定调用 Tool
→ Tool Executor
→ 执行 Tool
→ 返回结果

安全机制：

allowlist
执行确认
沙盒限制

------------------------------------------------------------
7 Workspace System（工作空间系统）
------------------------------------------------------------

每个 Agent 有独立 Workspace。

作用：

保存 Agent 状态
保存记忆
保存技能
保存日志

目录结构：

workspace/

USER.md
SOUL.md
MEMORY.md

skills/
memory/
logs/

特点：

Agent 运行环境隔离
可迁移
可备份
可独立扩展

------------------------------------------------------------
四、核心运行流程
------------------------------------------------------------

完整消息流程：

用户输入

↓

Web UI 或 CLI

↓

Gateway Runtime 接收消息

↓

Session Manager 找到对应 Agent

↓

Agent Runtime 启动 Agent Loop

↓

Context Builder 构建 Prompt

↓

调用 LLM

↓

LLM 判断是否调用 Tool

↓

如果调用 Tool

执行 Tool
返回结果

↓

更新 Memory

↓

生成最终回复

↓

Gateway 返回结果

↓

Web UI 显示回复
